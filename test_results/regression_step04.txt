=== IMPLEMENTATION NOTES ===

NLBar.tsx:
- Ghost text uses a layered approach: transparent span for the typed prefix, dim span
  for the synopsis suffix. Only shown when the hint starts with what the user has typed.
- OllamaUnavailableError recovery panel preserves the query; each command line is a
  copyable button.
- nlQuery is forwarded through setSuggestion so the Block can be labeled correctly.

SuggestionCard.tsx:
- Promise.allSettled([validateCommand, explainCommand]) runs on mount.
- Re-validation on blur fires only when editedSinceValidate is true AND the previous
  result was warn or block — avoids unnecessary model calls.
- Confusable highlight splits the command string into ASCII/non-ASCII segments and
  renders non-ASCII chars in amber with a wavy underline.
- "Run anyway" label: shown when explanation.reversible === false AND verdict is not block.
- validateCommand fail-open: if the validator throws, the card shows 'warn' rather than
  blocking (spec §12).

Terminal.tsx:
- Config fetched from main process once on mount; DEFAULT_CONFIG used as initial state
  so components render immediately without waiting.
- Sidebar is a Phase 4 placeholder that shows the Explain result. Full sidebar in Phase 5.
- handleConfirm appends '\r' (not '\n') before ptyWrite — PTY_WRITE strips after '\n',
  and '\r' is the PTY's Enter signal.
- The Help button in the input area and the Help button on TerminalBlock both route
  through handleHelp(), which calls explainResult(includeOutput: true).

=== KNOWN GAPS (acceptable at MHTP) ===
- Sidebar is a placeholder: Navigate, Glossary, Settings tabs are non-functional
  (labeled but disabled). Phase 5 delivers the full sidebar.
- No command history in NLBar (deferred to v2 per spec §17).
- Config persistence changes (font size, theme) not reflected in xterm until restart.
  Phase 6 wires config IPC updates to live xterm options.
- The ghost text only shows the suffix when the synopsis starts with the typed value.
  Edge case: user types "find . -name" → synopsis "find [path] [expression]" diverges
  after "find ", so no suffix shown. The spec example shows this case as "still shown"
  which requires a different rendering approach. The current implementation correctly
  shows ghost text for the common case (typing the command name) and is silent when
  the user has typed beyond the synopsis. Acceptable for MHTP.
