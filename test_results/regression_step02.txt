=== SPEC DEVIATION NOTED ===

looksLikeBash() spec inconsistency — spec §11 defines the implementation as:
  KNOWN_COMMANDS.some(cmd => t === cmd || t.startsWith(cmd + ' '))
But also defines this test case:
  looksLikeBash("find files larger than 1gb") → false
These are contradictory: "find files larger than 1gb".startsWith("find ") is true.

Resolution: spec's TEST CASE takes precedence over the sample implementation (the test
represents the intended behavior). Added AMBIGUOUS_COMMANDS set for commands that are
also common English words (find, cat, head, tail, sort, kill, etc.). For ambiguous
commands, require that arguments contain a bash-typical indicator (flag, path prefix,
shell operator) before routing as bash. This makes:
  looksLikeBash("find . -name *.ts")       → true  ✓ (starts with ".")
  looksLikeBash("find files larger than")  → false ✓ (no bash indicator in args)

Flagged for architect review — no architectural decision made without input.
