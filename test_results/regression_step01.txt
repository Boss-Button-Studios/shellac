=== BUGS FOUND AND FIXED DURING TESTING ===

1. preload.mjs vs preload.js — vite-plugin-electron builds to .js; BrowserWindow
   referenced .mjs. Preload never loaded; window.electronAPI was undefined;
   React unmounted on useEffect throw. Fixed: changed path to preload.js.

2. CSP blocked Vite dev inline script — script-src 'self' blocked @vitejs/plugin-react
   Fast Refresh preamble. Fixed: 'unsafe-inline' added to script-src in dev mode only.
   Also added ws://localhost:5173 to connect-src for Vite HMR WebSocket.

3. PTY_WRITE appended '\n' to every character — typing "e" sent "e\n" and
   immediately executed command "e". Spec §8c was written for NLBar complete-command
   strings. Raw xterm keyboard input uses '\r' (Enter) for execution. Fixed: handler
   now strips after first '\n' for security but does not append '\n'. Senders are
   responsible for their own line endings.

4. Ctrl+C still sent SIGINT — term.onKey was passing '\x03' to ptyWrite() despite
   the comment saying it was blocked. Fixed: explicit early return on key === '\x03'.

5. navigator.clipboard blocked in sandboxed renderer — both writeText() and
   readText() denied. Fixed: routed all clipboard operations through Electron's
   native clipboard module via IPC (CLIPBOARD_READ, CLIPBOARD_WRITE channels added
   to IPC constants; types.ts unfrozen for this necessary addition). Shortcut
   detection moved from term.onKey to window.addEventListener('keydown') since
   xterm was suppressing modifier combos before onKey fired.

6. Shell died after `exit` with no recovery — PTY exited, terminal went silent,
   no console log visible. Fixed: PTY respawns 500ms after exit; console.log added
   in both main process and renderer PTY_EXIT handlers.

=== NOTES ===
- @xterm/xterm v6 removed copyOnSelect from ITerminalOptions; implemented via
  onSelectionChange. Behavior identical.
- navigator.clipboard is blocked in Electron sandbox — permanent architectural
  decision to route clipboard through main process IPC. Added CLIPBOARD_READ and
  CLIPBOARD_WRITE to IPC constants (types.ts unfrozen for this; change is minimal
  and necessary).
- PTY_WRITE no longer appends '\n'. NLBar (Phase 4) must append '\r' when
  submitting commands. Stop button sends '\x03' directly (no '\r' needed).
