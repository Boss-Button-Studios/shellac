[BlockStore logic — inline Node.js test]
Three assertions verified programmatically:

  PASS: startBlock + appendOutput + finishBlock round-trip
    - startBlock() creates block with correct shape
    - appendOutput() accumulates chunk into block.output
    - finishBlock() sets exitCode and finishedAt
    - block found in store with correct values

  PASS: block cap drops oldest (cap=3, added 4, first is b1)
    - After 4 insertions with cap=3, array length is 3
    - Oldest block (b0) dropped, b1 is now first
    - Newest block (b3) present at end

  PASS: getContextBlocks excludes active and flagged blocks
    - Active block (finishedAt: null) excluded
    - Flagged block (contextFlagged: true) excluded
    - Two finished non-flagged blocks returned

[PTY shell spawn — code inspection]
spawnPty() in electron/main.ts:
  - pty.spawn(config.shell, [], { name: 'xterm-256color', cwd: os.homedir() }) ✓
  - CWD hook injected for 'zsh' and 'bash' shells ✓
  - onData: CWD_REGEX strips OSC sequences, sends PTY_DATA (clean) and PTY_CWD ✓
  - onExit: sends PTY_EXIT with exitCode ✓

[PTY_WRITE control character handling — code inspection]
  - Single-byte control chars (e.g. \x03) written directly, no '\n' appended ✓
  - Multi-byte input: split('\n')[0] + '\n' (spec §8c) ✓

[Electron application menu — code inspection]
  - buildAppMenu() sets Edit submenu: Copy / Paste / Select All
  - macOS: Cmd+C / Cmd+V / Cmd+A
  - Linux: Ctrl+Shift+C / Ctrl+Shift+V / Ctrl+Shift+A
  - Ctrl+C intentionally absent from menu (Stop button is SIGINT path) ✓

[xterm copyOnSelect — code inspection]
  - term.onSelectionChange() → clipboard.writeText(term.getSelection())
  - Note: @xterm/xterm v6 removed the copyOnSelect option from ITerminalOptions.
    Implemented via event handler instead. Behavior is identical. ✓

[Keyboard clipboard shortcuts — code inspection]
  - Cmd+C (mac) / Ctrl+Shift+C (linux): getSelection() → clipboard.writeText() ✓
  - Cmd+V (mac) / Ctrl+Shift+V (linux): clipboard.readText() → ptyWrite() ✓
  - Ctrl+C: NOT wired to SIGINT ✓

[Stop button — code inspection]
  - Visible when activeBlockId !== null ✓
  - Hidden when activeBlockId === null; NLBar placeholder shown ✓
  - onClick: ptyWrite('\x03') ✓
  - \x03 is a single-byte control char — passes through PTY_WRITE handler
    without newline appended (per fix above) ✓

[TerminalBlock structure — code inspection]
  - ansi-to-html instantiated with escapeXML: true (non-negotiable) ✓
  - Header: $ glyph + command + exit badge (✓ 0 / ✗ N) + elapsed time ✓
  - nlQuery label rendered above header when present ✓
  - Output area: border-left amber, ansi→html with XSS protection ✓
  - Truncation at 100 lines with "Show all N lines" toggle ✓
  - Click header: copies command to clipboard ✓
  - "Help me fix this" button: visible on non-zero exit, disabled (stub for Phase 4) ✓
